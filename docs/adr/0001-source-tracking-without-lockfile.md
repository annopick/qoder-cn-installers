# 使用来源追踪文件而非锁文件

不采用参考实现（vercel-labs/skills）的全局+项目级双锁文件方案，改为在 `~/.qoder-cn/.qci.source.json` 中按资源类型分组记录每个已安装资源的来源仓库和 ref。

原因：QCI 只有用户级安装场景，没有项目级需求。已安装状态可以通过直接扫描目标目录（`~/.qoder-cn/skills/`、`~/.qoder-cn/agents/`、解析 `mcp.json`）获得，无需锁文件做完整快照。来源追踪文件只解决"这个资源从哪来"这一个问题，保持最小职责。

**Considered Options**:
- 全局+项目级双锁文件（参考实现方案）——过度设计，项目级场景不存在
- 每个资源目录内嵌 `.source.json`——文件分散，`list` 和 `update` 需要遍历多个目录
- 单一来源追踪文件（已选择）——集中管理，一个文件即可支撑 `update` 和 `remove`
